Stochastic Gradient Descent (SGD) — Complete Guide

Term Definitions​

Before diving in, understand the three words in the name:

TermDefinition
StochasticRandomly determined — has a random probability distribution that can be analysed statistically but cannot be predicted precisely
GradientThe vector of partial derivatives of a function with respect to all its variables
DescentMoving downward toward the minimum value

Combined definition:

Stochastic Gradient Descent is an optimization algorithm where the gradient is calculated using only ONE randomly selected data point at each iteration, rather than the entire dataset.

1. The Problem with Batch Gradient Descent​

Before understanding SGD, you must understand why Batch GD breaks down. There are two concrete problems.

Problem 1 — Computational Complexity​

In Batch GD, to calculate the coefficients you must compute the derivative of the loss function with respect to each parameter using all rows in the dataset:

∂J/∂βⱼ = −(2/m) Σᵢ₌₁ᵐ (yᵢ − ŷᵢ) · xᵢⱼ

Where m = number of rows (samples).

The computational cost explodes with data size:

Scenario A — Small dataset:

ParameterValue
Rows (n)1,000 → 10³
Columns5 → 6 coefficients → ~10²
Epochs50 → ~10³
Total derivatives10³ × 10² × 10³ = 10⁸

Scenario B — Large dataset (big data):

ParameterValue
Rows (n)100,000 → 10⁵
Columns100 → ~10² coefficients
Epochs1,000 → 10³
Total derivatives10⁵ × 10² × 10³ = 10¹⁰
Warning

With 10¹⁰ calculations, Batch GD becomes extremely slow on big data — and this only gets worse as you scale.

Problem 2 — Memory / Vectorization Bottleneck​

Batch GD uses vectorization to avoid loops and speed up computation:

y_hat = np.dot(X_train, self.coef_) + self.intercept_

This single line computes predictions for all rows simultaneously — but it requires loading the entire dataset into RAM at once.

If your dataset is 1 GB, 10 GB, or larger — it simply doesn't fit. The result: memory errors and crashes.

Where Batch GD Fails in Practice​

DomainData TypeWhy Batch GD Fails
CNNsImagesHigh-dimensional data, millions of pixels per image
RNNsTextLarge vocabularies, long sequences
Deep LearningMixedMassive parameter space, enormous datasets

2. Stochastic Gradient Descent — Core Concept​

The Key Idea​

Instead of using all m data points to compute the gradient before updating parameters, SGD uses one randomly selected data point per update.

AspectBatch GDStochastic GD
Data used per updateAll m rows1 random row
Updates per epoch1m (one per row)
Convergence speedSlower overallFaster overall
Memory requirementHigh (entire dataset)Low (one row at a time)
Solution stabilityStable, consistentVariable, fluctuating
Noise in updatesNoneHigh

Why "Stochastic"?​

The row selected for each parameter update is chosen randomly. This introduces randomness (stochasticity) into the optimization process. As a direct consequence:

  • Running SGD twice on the same data gives different results
  • The path to the minimum is noisy and zigzag, not smooth
  • This noise is actually useful — it helps escape local minima (more on this later)

3. Mathematical Formulation​

The Loss Function (MSE)​

J(β₀, β₁, …, βₙ) = (1/m) Σᵢ₌₁ᵐ (yᵢ − ŷᵢ)²

Where:

ŷᵢ = β₀ + β₁xᵢ₁ + β₂xᵢ₂ + … + βₙxᵢₙ

Batch GD Update Rule (Reference)​

Uses all m samples at every step:

Intercept:

∂J/∂β₀ = −(2/m) Σᵢ₌₁ᵐ (yᵢ − ŷᵢ)

β₀ = β₀ − α · ∂J/∂β₀

Coefficient for feature j:

∂J/∂βⱼ = −(2/m) Σᵢ₌₁ᵐ (yᵢ − ŷᵢ) · xᵢⱼ

βⱼ = βⱼ − α · ∂J/∂βⱼ

SGD Update Rule​

Uses ONE randomly selected sample at every step:

Intercept:

∂J/∂β₀ = −2 · (yᵢ − ŷᵢ)

β₀ = β₀ − α · (−2)(yᵢ − ŷᵢ)
   = β₀ + 2α · (yᵢ − ŷᵢ)

Coefficient for feature j:

∂J/∂βⱼ = −2 · (yᵢ − ŷᵢ) · xᵢⱼ

βⱼ = βⱼ − α · (−2)(yᵢ − ŷᵢ) · xᵢⱼ
   = βⱼ + 2α · (yᵢ − ŷᵢ) · xᵢⱼ

Key Mathematical Differences​

Formula ComponentBatch GDStochastic GD
SummationΣᵢ₌₁ᵐ (all rows)Single row only
Division by m1/m presentNot needed
Data points usedAll m points1 random point
Prediction shapeVector Ŷ (m×1)Scalar ŷᵢ
Note

The two key simplifications in SGD vs Batch GD: the sum collapses to a single term, and the 1/m normalisation disappears — because there is only one sample.

Step-by-Step Gradient Calculation for SGD​

Given a randomly selected index idx:

Single row:  X_idx = [x_idx,1 , x_idx,2 , …, x_idx,n]
Actual value: y_idx

Step 1 — Compute prediction:

ŷ_idx = β₀ + Σⱼ βⱼ · x_idx,j

Step 2 — Compute error:

error = y_idx − ŷ_idx

Step 3 — Update intercept:

β₀ = β₀ − α · (−2 · error)
   = β₀ + 2α · error

Step 4 — Update all coefficients:

βⱼ = βⱼ − α · (−2 · error · x_idx,j)
   = βⱼ + 2α · error · x_idx,j

5. Time Complexity Analysis​

A common question: if epochs are fixed, which is faster?

Answer: For a fixed number of epochs, Batch GD does fewer total updates:

AlgorithmUpdates per epochTotal updates (100 epochs)
Batch GD1100
SGDn (rows)100 × n

At first glance, Batch GD looks cheaper. But this misses the bigger picture:

Why SGD is Still Preferred Overall​

SGD needs far fewer epochs to converge:

ScenarioBatch GDSGD
Epochs needed to converge1,00010
Updates per epoch1353
Total updates1,0003,530
Wall-clock timeLongerShorter
Key insight

SGD converges faster in total time because it reaches a good solution in far fewer epochs, even though it does more updates per epoch.

6. Implementation from Scratch​

Dataset Setup​

from sklearn.datasets import load_diabetes
from sklearn.model_selection import train_test_split
from sklearn.metrics import r2_score
import numpy as np

X, y = load_diabetes(return_X_y=True)
print(f"Dataset shape: {X.shape}")   # (442, 10)

X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=2
)

Batch GD Implementation (Reference)​

class GDRegressor:
    """Batch Gradient Descent Regressor"""

    def __init__(self, learning_rate=0.01, epochs=100):
        self.coef_      = None
        self.intercept_ = None
        self.lr         = learning_rate
        self.epochs     = epochs

    def fit(self, X_train, y_train):
        self.intercept_ = 0
        self.coef_      = np.ones(X_train.shape[1])

        for i in range(self.epochs):
            # Predict for ALL rows at once (vectorized)
            y_hat = np.dot(X_train, self.coef_) + self.intercept_

            # Intercept gradient: −(2/m) Σ(yᵢ − ŷᵢ)
            intercept_der   = -2 * np.mean(y_train - y_hat)
            self.intercept_ = self.intercept_ - (self.lr * intercept_der)

            # Coefficient gradients: −(2/m) Xᵀ(Y − Ŷ)
            coef_der   = -2 * np.dot((y_train - y_hat), X_train) / X_train.shape[0]
            self.coef_ = self.coef_ - (self.lr * coef_der)

    def predict(self, X_test):
        return np.dot(X_test, self.coef_) + self.intercept_

SGD Implementation​

class SGDRegressor:
    """Stochastic Gradient Descent Regressor — from scratch"""

    def __init__(self, learning_rate=0.01, epochs=100):
        self.coef_      = None
        self.intercept_ = None
        self.lr         = learning_rate
        self.epochs     = epochs

    def fit(self, X_train, y_train):
        self.intercept_ = 0
        self.coef_      = np.ones(X_train.shape[1])

        for i in range(self.epochs):
            for j in range(X_train.shape[0]):

                # RANDOMLY select ONE index
                idx = np.random.randint(0, X_train.shape[0])

                # Predict for that SINGLE row → scalar, not vector
                y_hat = np.dot(X_train[idx], self.coef_) + self.intercept_

                # Intercept gradient: −2(yᵢ − ŷᵢ)   [no mean, single value]
                intercept_der   = -2 * (y_train[idx] - y_hat)
                self.intercept_ = self.intercept_ - (self.lr * intercept_der)

                # Coefficient gradient: −2(yᵢ − ŷᵢ) · xᵢ   [no mean, single row]
                coef_der   = -2 * np.dot((y_train[idx] - y_hat), X_train[idx])
                self.coef_ = self.coef_ - (self.lr * coef_der)

        print(f"Intercept:    {self.intercept_}")
        print(f"Coefficients: {self.coef_}")

    def predict(self, X_test):
        return np.dot(X_test, self.coef_) + self.intercept_

Code Differences — Batch GD vs SGD​

Code ElementBatch GDSGD
Inner loopNonefor j in range(X_train.shape[0])
Row selectionAll rows (vectorized)idx = np.random.randint(0, X_train.shape[0])
y_hat shapeVector (m,)Scalar
Intercept derivativenp.mean(y_train - y_hat)y_train[idx] - y_hat
Coefficient derivativenp.dot(error, X_train) / mnp.dot(error, X_train[idx])

Training and Results​

import time

# Batch GD
start = time.time()
gdr = GDRegressor(epochs=1000, learning_rate=0.5)
gdr.fit(X_train, y_train)
gdr_pred = gdr.predict(X_test)
print(f"Batch GD  | R²: {r2_score(y_test, gdr_pred):.4f} | Time: {time.time()-start:.4f}s")

# SGD
start = time.time()
sgd = SGDRegressor(learning_rate=0.01, epochs=40)
sgd.fit(X_train, y_train)
sgd_pred = sgd.predict(X_test)
print(f"SGD       | R²: {r2_score(y_test, sgd_pred):.4f} | Time: {time.time()-start:.4f}s")

8. When to Use SGD​

Use Case 1 — Big Data​

ScenarioRecommendation
n < 10,000 rowsBatch GD is fine
n > 100,000 rowsUse SGD
n > 1,000,000 rowsSGD is mandatory
Deep Learning (images/text)Always SGD or Mini-Batch

Use Case 2 — Non-Convex Cost Functions​

Convex function — single global minimum, no local minima. Batch GD works well.

Non-convex function — multiple local minima, one global minimum. Batch GD can get stuck in a local minimum and never find the global one.

SGD's random noise helps it jump out of local minima and continue searching for the global minimum.

AlgorithmCost FunctionSGD Benefit
Logistic RegressionNon-convexEscapes local minima
SVMNon-convexBetter generalisation
Neural NetworksHighly non-convexEssential for training
Deep LearningVery non-convexStandard approach

9. Learning Rate Schedules​

The Problem with a Fixed Learning Rate​

When SGD gets close to the solution, random oscillations keep it jumping around instead of settling. A fixed learning rate that's good for early training is too large for fine-grained convergence at the end.

The Solution — Learning Rate Decay​

Concept: Start with large steps (fast exploration), decrease the learning rate as training progresses (fine-grained settling).

Initial: large steps → Later: small steps

Time-Based Decay Formula​

def learning_rate(lr_initial, total_iterations, current_iteration):
    """
    Time-based decaying learning rate.
    lr_initial       : starting learning rate (η₀)
    total_iterations : epochs × rows
    current_iteration: current step i
    """
    return lr_initial / (1 + (current_iteration / total_iterations))

How It Decays in Practice​

Example: η₀ = 0.1, total iterations t = 100

00.000.100

750.750.057

Types of Learning Rate Schedules​

ScheduleFormulaDescription
Constantη = η₀No decay, fixed rate
Time-basedη = η₀ / (1 + d×t)Linear decay
Step decayη = η₀ × dᶠˡᵒᵒʳ⁽ᵗ/ʳ⁾Drops by factor d every r epochs
Exponentialη = η₀ × e^(−dt)Exponential decay
AdaptiveVariesAdam, RMSprop, Adagrad, etc.

SGD with Learning Rate Schedule — Full Implementation​

class SGDRegressorWithSchedule:
    """SGD with time-based learning rate decay."""

    def __init__(self, learning_rate=0.01, epochs=100):
        self.coef_      = None
        self.intercept_ = None
        self.lr_initial = learning_rate
        self.epochs     = epochs

    def _get_lr(self, iteration, total_iterations):
        """Time-based decay: η = η₀ / (1 + i/t)"""
        return self.lr_initial / (1 + (iteration / total_iterations))

    def fit(self, X_train, y_train):
        self.intercept_   = 0
        self.coef_        = np.ones(X_train.shape[1])
        total_iterations  = self.epochs * X_train.shape[0]
        iteration         = 0

        for epoch in range(self.epochs):
            for j in range(X_train.shape[0]):

                # Decay learning rate at every step
                lr  = self._get_lr(iteration, total_iterations)

                idx = np.random.randint(0, X_train.shape[0])
                y_hat = np.dot(X_train[idx], self.coef_) + self.intercept_

                intercept_der   = -2 * (y_train[idx] - y_hat)
                self.intercept_ = self.intercept_ - (lr * intercept_der)

                coef_der   = -2 * np.dot((y_train[idx] - y_hat), X_train[idx])
                self.coef_ = self.coef_ - (lr * coef_der)

                iteration += 1

    def predict(self, X_test):
        return np.dot(X_test, self.coef_) + self.intercept_

Key Takeaways​

#Takeaway
1SGD uses one random sample per update instead of the entire dataset
2This produces n updates per epoch vs 1 for Batch GD
3Despite more updates per epoch, SGD converges faster overall (fewer epochs needed)
4Low memory — never needs the entire dataset in RAM simultaneously
5Results vary between runs due to random row selection
6Can escape local minima in non-convex loss landscapes — crucial for deep learning
7Learning rate schedules (decay) improve convergence stability near the optimum
8Essential for deep learning, NLP, computer vision — any domain with big data
