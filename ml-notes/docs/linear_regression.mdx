# Linear Regression

A supervised ML algorithm that finds the best-fit straight line through data,
then uses that line to predict new values.

---

## The Equation

```
y = m·x + b
```

| Symbol | Name | Meaning |
|--------|------|---------|
| `y` | output | what we're predicting (e.g. house price) |
| `x` | input | the feature we know (e.g. house size) |
| `m` | weight / slope | how much y changes per unit of x |
| `b` | bias / intercept | baseline value when x = 0 |

With **multiple features**: `y = m₁x₁ + m₂x₂ + ... + mₙxₙ + b`

## How It Learns — Gradient Descent

The algorithm finds the best weights by repeatedly asking:
*"Which direction reduces my error?"* and taking a small step that way.

```
new_weight = old_weight − learning_rate × gradient
```

- **Learning rate too high** → overshoots the minimum, bounces around
- **Learning rate too low** → converges, but very slowly
- **Gradient = 0** → we've found the minimum (converged)

> Try stepping manually, or hit **Run** to watch it converge. Crank the learning rate above 0.7 to see it overshoot.

## Multiple Features

With more than one input, each feature gets its own weight:

```
price = m₁×size + m₂×rooms + m₃×age + b
```

The learning algorithm is identical — gradient descent now updates all weights simultaneously.

```python
from sklearn.linear_model import LinearRegression
import numpy as np

# X has multiple columns — one per feature
X = np.array([
    [75, 3, 20],   # [size_m2, rooms, age_years]
    [50, 2, 35],
    [100, 4, 5],
])
y = np.array([403, 260, 580])  # price in $k

model = LinearRegression()
model.fit(X, y)

print(model.coef_)       # [m1, m2, m3]
print(model.intercept_)  # b

model.predict([[80, 3, 15]])
```

**Negative weights** are normal — a higher age year *decreases* predicted price, so `m₃` will be negative.

## Evaluation Metrics

| Metric | Formula | Meaning |
|--------|---------|---------|
| **R²** | `1 − SS_res / SS_tot` | % of variance explained. 1 = perfect, 0 = useless |
| **MAE** | `mean( \|actual − predicted\| )` | Average error in same units as output |
| **RMSE** | `sqrt( mean( (actual − predicted)² ) )` | Like MAE but punishes large errors harder |

```python
from sklearn.metrics import r2_score, mean_absolute_error, mean_squared_error

r2   = r2_score(y_test, y_pred)
mae  = mean_absolute_error(y_test, y_pred)
rmse = mean_squared_error(y_test, y_pred, squared=False)
```

**Always evaluate on a held-out test set** — never on the data you trained with.

## Key Concepts

### Overfitting
The model memorises training data (including noise) and performs poorly on new data.

Fix with:
- More training data
- **Ridge regression** — adds `λ · Σm²` penalty to loss (L2 regularisation)
- **Lasso regression** — adds `λ · Σ|m|` penalty, drives some weights to zero (L1)

### Feature Scaling
When features have very different ranges (size in m² vs age in years), gradient descent struggles. Normalise first:

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

### Train / Test Split
```python
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
```

---

## Assumptions

Linear regression works best when:

1. The true relationship is actually linear
2. Residuals (errors) are normally distributed
3. No strong correlation between features (multicollinearity)
4. Variance of errors is constant (homoscedasticity)

---

## When to Use It

| ✓ Good fit | ✗ Poor fit |
|-----------|-----------|
| Predicting continuous values | Classification problems |
| Need interpretable weights | Non-linear relationships |
| Small to medium datasets | Very high-dimensional sparse data |
| Quick baseline model | Complex pattern recognition |

> **Rule of thumb:** always start with linear regression as your baseline. If it performs well enough, ship it — simpler models are easier to debug, explain, and maintain.
